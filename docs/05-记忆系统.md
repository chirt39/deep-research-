# 05 — 记忆系统

> 记忆系统实现跨会话的上下文保留。用户上次问了"比亚迪的 ROE"，这次问"估值呢"，系统能自动关联。

---

## 5.1 架构概览

```
                    MemoryManager（统一入口）
                    /        |         \
             短期记忆    长期记忆(语义)   长期记忆(情景)
           /     |      /      \        /      \
      Redis  PostgreSQL SQLite Milvus SQLite PostgreSQL
      (内存)  (磁盘)    (磁盘)  (向量)  (磁盘)  (磁盘)
```

### 双层记忆设计

| 层级 | 存储内容 | 生命周期 | 后端选择 |
|------|----------|----------|----------|
| **短期记忆** | 当前对话的原始消息（最近 N 轮） | TTL 7 天 | Redis / PostgreSQL / 内存 |
| **长期记忆-语义** | 用户偏好、事实知识 | 永久 | PostgreSQL + Milvus / SQLite |
| **长期记忆-情景** | 历史研究任务记录 | 永久 | PostgreSQL / SQLite |

```python
class MemoryType(Enum):
    SHORT_TERM = "short_term"    # 对话消息
    SEMANTIC = "semantic"        # 事实、偏好
    EPISODIC = "episodic"        # 任务记录

@dataclass
class MemoryEntry:
    id: str                      # 唯一标识（UUID）
    content: Any                 # 记忆内容（dict/str）
    memory_type: MemoryType      # 类型
    user_id: str                 # 所属用户
    thread_id: Optional[str]     # 所属会话（可选）
    namespace: Optional[str]     # 分类命名空间（如 "facts/投资偏好"）
    metadata: Dict[str, Any]     # 元数据（tenant_id, created_at 等）
    created_at: datetime
```

```python
class ShortTermMemory:
    """内存版短期记忆实现。
    
    内部结构：
    {
      thread_id: {
        "messages": [BaseMessage, ...],   # 原始消息列表
        "metadata": {Dict},               # 线程元数据
      }
    }
    
    核心方法：
    - add_message(thread_id, message, metadata)    → 追加消息
    - get_messages(thread_id, include_summary, last_n) → 获取最近 N 条
    - clear_thread(thread_id)                       → 清除线程
    - update_thread_metadata(thread_id, metadata)   → 更新元数据
    
    内存实现特点：
    - 进程重启后数据丢失
    - 生产环境建议用 Redis 或 PostgreSQL 后端
    """
```

### 5.5.2 短期记忆操作

```python
def add_short_term_message(self, thread_id, message, metadata, user_id, tenant_id):
    """追加一条消息到短期记忆。
    
    按后端分发：
    - Redis: RPUSH + EXPIRE + 触发压缩检查
    - PostgreSQL: INSERT + 触发压缩检查
    - 内存: ShortTermMemory.add_message()
    """

def get_short_term_messages(self, thread_id, include_summary, last_n, user_id, tenant_id):
    """获取最近 N 条消息。
    
    如果 include_summary=True 且有历史摘要：
    → 返回 [SystemMessage("历史对话摘要：..."), *recent_messages]
    """

def _compress_redis_thread(self, tenant_id, user_id, thread_id):
    """Redis 短期记忆压缩。
    
    触发条件：消息数 > short_term_max_messages（默认 30）
    压缩策略：
    ① 保留最近 short_term_summary_threshold 条（默认 20）
    ② 其余消息发送给摘要 LLM → 生成 100-300 字摘要
    ③ 摘要存入独立 key（summary:{tenant}:{user}:{thread}）
    ④ 旧消息删除
    """

def _summarize_text(self, existing_summary, history_slice):
    """对话摘要生成。
    
    预加载摘要 LLM：
    → build_summary_llm(model="deepseek-chat", temperature=0.1)
    
    模板：
    "你是对话压缩引擎。保留事实、偏好、结论、待办和约束。
     已有摘要：{existing_summary or '无'}
     新增历史：{history_text}
     输出：100-300 字中文摘要"
    """
```

### 5.5.5 记忆持久化

```python
def persist_turn(self, tenant_id, user_id, thread_id, query, answer):
    """每轮对话结束后调用，将本轮问答持久化到所有记忆层。
    
    流程：
    ① 短期记忆：追加 user 消息 + assistant 消息
    ② 长期记忆提取：
       - 检测"记住/偏好/我叫"等关键词
       - 如有 → extract_memory_from_messages() → facts + preferences
       - 自动提取的用户事实/偏好写入长期记忆
    ③ 任务记忆（如果 save_conversation_task=True）：
       save_task(type="conversation", data={query}, outcome=answer)
    
    注意：只有含"记住/偏好"等关键词的对话才提取长期记忆，
    避免将无关闲聊写入永久存储。
    """
```

## 5.7 记忆数据流

```
用户输入："比亚迪002594估值分析"
     │
     ├── build_personalized_prompt_context()
     │   ├─ Milvus 检索 → 找到"用户偏好 DCF 估值"
     │   ├─ 短期记忆 → 找到上次问的"比亚迪 ROE"
     │   └─ 组装注入块 → 返回给 main.run_query()
     │
     ├── 注入块拼接到所有节点 Prompt 前面
     │   "请分析比亚迪估值 [跨会话记忆] 用户偏好DCF..."
     │
     ├── 工作流执行...
     │
     └── persist_turn()
         ├─ 短期记忆：追加本轮 Q&A
         └─ 长期记忆：如果用户说了"记住..." → 提取并存储
```
