### 核心设计原则

| 原则 | 实现方式 |
|------|----------|
| **多智能体分工** | 9 个独立 Agent，每个有专属 System Prompt，通过 LangGraph 编排 |
| **证据驱动** | 所有结论绑定 `source_id`，支持溯源 |
| **双源并行** | 网络搜索 + 本地知识库同时执行，互不阻塞 |
| **迭代深挖** | Analyst 发现信息缺口 → Reflect 生成补搜计划 → 回到检索 |
| **跨会话记忆** | 短期（对话上下文）+ 长期（Milvus 向量存储），自动注入 Prompt |
| **降级容错** | Bocha 未配 → DuckDuckGo；AKShare 未装 → stub；Milvus 为空 → 跳过 |

## 1.2 数据流全景

一次完整的"比亚迪深度研究"请求，数据在各组件间的流转：

```
① 用户输入 "深度研究比亚迪002594"
     │
② intent_node → detect_intent() 规则匹配 "研究/分析/估值" → multiagent
     │                              ↑ 金融关键词直接跳，不调 LLM
③ plan_node → LLM 拆解为 5 个子问题 + 搜索计划
     │          宏观/行业/公司/估值/舆情 各 1-2 个问题
     │
④ ┌─ web_search_node (并行) ─────────────────────┐
   │   ├─ bocha_web_search_records                  │
   │   │   └─ BOCHA_API_KEY? → Bocha API            │
   │   │   └─ 未配? → DuckDuckGo 免费搜索            │
   │   ├─ collect_finance_data_records              │
   │   │   └─ detect 002594 → AKShare:              │
   │   │       ├─ 新闻 (stock_news_em)               │
   │   │       ├─ 财务 (stock_financial_abstract_ths)│
   │   │       └─ 公司信息 (stock_individual_info)   │
   │   └─ LLM 过滤 → web_evidence[]                 │
   │                                                 │
   └─ local_rag_node (并行) ───────────────────────┘
       └─ search_knowledge_base_records
           └─ Milvus 向量检索 → local_evidence[]
     │
⑤ deep_dive_node → 证据评分 + 去重 + 冲突审计
     │              输入: web_evidence[] + local_evidence[]
     │              输出: evidence_pool[] + source_index[] + audit_flags[]
     │
⑥ analyze_node → 五维投资分析
     │             输入: evidence_pool[] + audit_flags[]
     │             输出: findings[] + needs_more_research?
     │
     ├─ needs_more_research=true → ⑦ reflect_node
     │                               生成补搜计划 → 回到④
     │
     └─ needs_more_research=false → ⑧ write_node
                                      整合 → Markdown 研报
                                      校验引用 → 拼接参考资料
     │
⑨ 返回 final (SSE 流式推送给前端)
```

## 1.3 配置系统

配置加载优先级：`.env 环境变量 > config.json > 代码默认值`

### 1.3.1 config.py — AppConfig 数据类

```python
# 文件: app/mult_agents/config.py
# 这是一个 frozen dataclass，实例化后不可修改
# 要改配置必须用 with_overrides() 创建新实例

@dataclass(frozen=True)
class AppConfig:
    # ═══ LLM 配置 ═══
    api_key: str              # DeepSeek API Key (sk-...)
    model: str                # deepseek-chat / deepseek-reasoner
    base_url: str             # https://api.deepseek.com/v1
    temperature: float        # 0.0~1.0，默认 0.3

    # ═══ 嵌入模型配置 ═══
    embedding_provider: str   # huggingface | openai | dashscope
    embedding_api_key: str    # 嵌入模型 API Key（可与 LLM Key 不同）
    embedding_model: str      # BAAI/bge-small-zh-v1.5 或 text-embedding-3-small
    embedding_base_url: str   # OpenAI 兼容嵌入 API 地址

    # ═══ 多租户 ═══
    thread_id: str            # 会话 ID（同一 thread 共享短期记忆）
    user_id: str              # 用户 ID（同一 user 共享长期记忆）
    tenant_id: str            # 租户 ID（数据隔离）

    # ═══ 工作流 ═══
    max_iterations: int       # 最大迭代轮次（默认 3）

    # ═══ 记忆系统 ═══
    enable_memory: bool       # 是否启用跨会话记忆
    short_term_ttl_seconds    # 短期记忆过期时间（默认 7 天）
    short_term_max_messages   # 触发压缩的阈值（默认 30 条）
    short_term_summary_threshold  # 压缩时保留最近 N 条（默认 20）
    short_term_backend: str   # postgres | redis | memory
    long_term_backend: str    # postgres | sqlite | disabled
    long_term_scope: str      # user（跨线程共享）| thread（线程隔离）
    save_conversation_task    # 是否把对话存为长期任务记忆
    checkpointer_backend: str # postgres | redis | memory | auto
    enable_milvus: bool       # 是否用 Milvus 做语义检索
    memory_top_k: int         # 每次注入的记忆条数（默认 6）

    # ═══ 基础设施 ═══
    redis_url: str            # Redis 连接（用于短期记忆/checkpointer）
    postgres_dsn: str         # PostgreSQL 连接
    milvus_host: str          # Milvus 地址
    milvus_port: int          # Milvus 端口（默认 19530）
    milvus_collection: str    # 记忆系统 Milvus 集合名
```

## 1.4 嵌入模型多后端设计

`llm_factory.py` 中的 `build_embeddings()` 函数支持 3 种嵌入后端，通过 `EMBEDDING_PROVIDER` 切换：

| provider | 模型 | 需要 API Key | 网络 | 适用场景 |
|----------|------|-------------|------|----------|
| `huggingface` | BAAI/bge-small-zh-v1.5 | 否 | 否 | **默认**，本地运行，中文效果好 |
| `openai` | text-embedding-3-small | 是 | 是 | 质量高，适合生产 |
| `dashscope` | text-embedding-v1 | 是 | 是 | 原项目兼容 |

同一个嵌入实例会被缓存（`use_cache=True`），避免重复创建。
