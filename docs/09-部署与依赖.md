## 9.3 启动方式

### 方式 1：完整前后端

```bash
# 终端 1 — 后端
cd d:/Downloads/deep_research-master/deep_research-master/deep_research
pip install -r requirements.txt
# 配置 config.json 中的 api_key
python -m uvicorn app.app_main:app --host 0.0.0.0 --port 8000 --reload

# 终端 2 — 前端
cd front/agent_front
npm install
npm run dev

# 浏览器打开 http://localhost:5173
```

### 方式 2：纯命令行模式

```bash
cd d:/Downloads/deep_research-master/deep_research-master/deep_research

# 单次查询
python -m app.mult_agents.main --once-query "比亚迪002594估值分析"

# 交互模式
python -m app.mult_agents.main
# 你: 比亚迪002594估值分析
# AI: ## 投资摘要 ...
# 你: /memory          ← 查看记忆状态
# 你: quit             ← 退出
```

### 方式 3：Docker

```bash
# Milvus（知识库 + 记忆需要）
docker run -d --name milvus -p 19530:19530 milvusdb/milvus:latest

# PostgreSQL（记忆系统需要，可选）
docker run -d --name postgres -p 5432:5432 \
  -e POSTGRES_PASSWORD=postgres postgres:16

# Redis（记忆系统需要，可选）
docker run -d --name redis -p 6379:6379 redis:7
```

## 9.4 环境变量全集

```bash
# ═══ LLM（必填） ═══
DEEPSEEK_API_KEY=sk-xxx           # DeepSeek API Key
DEEPSEEK_MODEL=deepseek-chat      # deepseek-chat | deepseek-reasoner
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_TEMPERATURE=0.3

# ═══ 嵌入模型 ═══
EMBEDDING_PROVIDER=huggingface    # huggingface | openai | dashscope
EMBEDDING_API_KEY=                # openai/dashscope 需要
EMBEDDING_MODEL=BAAI/bge-small-zh-v1.5
EMBEDDING_BASE_URL=

# ═══ 可选 API Keys ═══
BOCHA_API_KEY=                    # Bocha 网页搜索（可选，不配则用 DuckDuckGo）

# ═══ 多租户 ═══
THREAD_ID=default
USER_ID=default_user
TENANT_ID=default_tenant

# ═══ 工作流 ═══
MAX_ITERATIONS=3

# ═══ 记忆系统 ═══
ENABLE_MEMORY=true                # false = 纯内存模式，不需要任何外部存储
SHORT_TERM_BACKEND=postgres       # postgres | redis | memory
LONG_TERM_BACKEND=postgres        # postgres | sqlite | disabled
LONG_TERM_SCOPE=user              # user | thread
CHECKPOINTER_BACKEND=auto         # postgres | redis | memory | auto
MEMORY_TOP_K=6

# ═══ 基础设施连接 ═══
REDIS_URL=redis://127.0.0.1:6379
POSTGRES_DSN=postgresql://127.0.0.1:5432/postgres
MILVUS_HOST=127.0.0.1
MILVUS_PORT=19530
MILVUS_COLLECTION=mult_agent_memory
```

## 9.5 最小化部署（零基础设施）

如果只想要核心功能（不关心跨会话记忆），最简配置：

```bash
# .env 文件只需一行
DEEPSEEK_API_KEY=sk-xxx

# config.json 中设置
"enable_memory": false

# 然后启动
pip install -r requirements.txt
python -m uvicorn app.app_main:app --host 0.0.0.0 --port 8000
```

这个模式下：
- ✅ DeepSeek LLM 正常工作
- ✅ DuckDuckGo 免费网页搜索
- ✅ AKShare 免费金融数据
- ✅ LangGraph 工作流正常编排
- ❌ 跨会话记忆（每次对话独立）
- ❌ 本地知识库（需要 Milvus）

## 9.6 常见问题

### Q: 启动报 `ModuleNotFoundError: No module named 'backend'`
**A:** 确认从项目根目录 `deep_research/` 启动，且使用 `app.app_main:app` 格式：
```bash
python -m uvicorn app.app_main:app --host 0.0.0.0 --port 8000
```

### Q: 参考资料显示"链接暂不可用"
**A:** 财务数据（同花顺）没有 URL。网页搜索和新闻有链接的会正常显示。

### Q: AKShare 报 "Remote end closed connection"
**A:** 你的网络环境拦截了 EastMoney API。同花顺数据源不受影响，但 PE/PB/市值数据无法获取。关闭代理可解决。

### Q: 过程太慢（>1 分钟）
**A:** 每次研究需要 6-7 次 LLM 调用（DeepSeek 每次 3-8s）。已经做了优化：意图路由免 LLM、证据压缩、空节点跳过。预估 25-35s 完成一次完整研究。

### Q: 如何只用命令行不用前端？
**A:** `python -m app.mult_agents.main --once-query "你的问题"`

### Q: 前端 dev 模式连不上后端
**A:** 检查 `vite.config.ts` 中 proxy 配置是否正确指向 `http://localhost:8000`。确认后端已启动。
