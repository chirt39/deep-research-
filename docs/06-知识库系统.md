# 06 — 知识库系统（RAG）

> 知识库系统基于 Milvus 向量数据库，支持将本地文档（研报、白皮书、年报等）转为可检索的知识。

---

## 6.1 RAGConfig（配置类）

```python
# 文件: app/mult_agents/rag/core.py

@dataclass(frozen=True)
class RAGConfig:
    # Milvus 连接
    milvus_host: str = "127.0.0.1"
    milvus_port: int = 19530
    collection_name: str = "mult_agent_knowledge"  # 独立于记忆系统的集合

    # 嵌入模型配置（与 config.py 的 embedding_provider 对齐）
    embedding_provider: str = "huggingface"    # huggingface | openai | dashscope
    embedding_api_key: str = ""
    embedding_model: str = "BAAI/bge-small-zh-v1.5"
    embedding_base_url: str = ""

    # 文档分块配置
    chunk_size: int = 500       # 每个 chunk 最大字符数
    chunk_overlap: int = 50     # 相邻 chunk 的重叠字符数
```

## 6.2 RAGSystem（核心类）

```python
class RAGSystem:
    """向量检索 + 文档入库的统一封装。
    
    每个实例绑定一个 Milvus 集合。
    同一集合可被多个 RAGSystem 实例共享（但嵌入模型需一致）。
    """

    def __init__(self, config=None):
        """初始化三步：
        
        ① 创建嵌入模型
           build_embeddings(provider, api_key, model, base_url)
           → 根据 provider 创建不同后端
           → huggingface: 首次运行会下载 BGE 模型（~400MB）
           → 后续使用缓存
        
        ② 创建文本分割器
           RecursiveCharacterTextSplitter(
               chunk_size=500,              # 每块 500 字
               chunk_overlap=50,            # 重叠 50 字（避免切断关键信息）
               separators=["\n\n","\n","。","！","？","；","，"," ",""]
               # 优先在段落/句子边界切割，其次在标点，最后在空格/字符
           )
        
        ③ 创建 Milvus 向量存储
           langchain_milvus.Milvus(
               embedding_function,           # 嵌入模型
               collection_name,              # 集合名
               connection_args={uri},        # Milvus 连接
               auto_id=True                  # 自动分配 ID
           )
        """

    # ── 检索接口 ──

    def search(self, query: str, k: int = 3) -> str:
        """文本化的检索结果（给 LLM 阅读）。
        
        格式：
        检索到的相关信息：
        1. {snippet}
           (来源: /docs/比亚迪年报.txt)
        2. {snippet}
           (来源: /docs/新能源研报.md)
        """

    def search_records(self, query: str, k: int = 5) -> list[dict]:
        """结构化的检索结果（给 nodes.py 使用）。
        
        流程：
        ① utility.has_collection(collection_name) → 集合不存在？ → 返回 []
        ② vectorstore.similarity_search(query, k=k)
           → embedding(query) → Milvus ANN 搜索 → 返回 top-k Document
        ③ 每条 Document 转为结构化记录：
           {
             "source_id": "LOC-{序号}",
             "doc_id": metadata.source（原始文件路径）,
             "title": 文件名,
             "snippet": page_content（文档片段）,
             "source_type": "local",
             "metadata": {...}
           }
        
        返回的记录直接进入 local_rag_node 的 raw_records。
        """

    # ── 入库接口 ──

    def add_documents(self, documents: list[Document]) -> int:
        """批量添加 Document 对象到 Milvus。"""

    def ingest_text(self, text: str, source: str) -> int:
        """入库单段文本。
        
        流程：
        ① text_splitter.create_documents([text], metadatas=[{"source": source}])
           → 将文本切分为多个 chunk
        ② add_documents(chunks)
           → 每个 chunk → embedding → 写入 Milvus
        ③ 返回 chunk 数
        """

    def ingest_paths(self, paths: Iterable[Path]) -> int:
        """批量入库文件。
        
        遍历每个文件：
        → 读取文本 → ingest_text() → 累加 chunk 数
        """
```

## 6.4 知识库与记忆系统的关系

```
Milvus 集合划分：
┌─────────────────────────────────┐
│ mult_agent_memory               │ ← 记忆系统
│  ├─ memory_type: semantic       │   用户画像/偏好/事实
│  ├─ memory_type: episodic       │   历史任务记录
│  └─ namespace: user_profile     │   用户配置
│      namespace: facts/...       │   用户事实
│      namespace: tasks/...       │   用户任务
├─────────────────────────────────┤
│ mult_agent_knowledge            │ ← 知识库系统
│  └─ source: /docs/比亚迪年报.txt │   存储的是文档 chunk
│     source: /docs/新能源研报.md  │   不是用户个人数据
└─────────────────────────────────┘

两者：
- 用不同的 collection → 数据隔离
- 共用一个 Milvus 实例 → 基础设施复用
- 共用同一个嵌入模型 → embedding 一致
```

## 6.5 典型使用流程

```bash
# 1. 启动 Milvus
docker run -d --name milvus -p 19530:19530 milvusdb/milvus:latest

# 2. 准备文档
mkdir D:/finance_docs
# 放入 .txt / .md / .pdf 文件

# 3. 入库
cd d:/Downloads/deep_research-master/deep_research-master/deep_research
python -m mult_agents.rag.ingest D:/finance_docs

# 输出：
# 待入库文件: 15 个
# 目标集合:   mult_agent_knowledge
# 嵌入模型:   huggingface/BAAI/bge-small-zh-v1.5
#   [1/15] 比亚迪2025年报.txt → 32 chunks
#   [2/15] 新能源行业策略.md → 18 chunks
#   ...
# 入库完成: 15 个文件, 420 个 chunk

# 4. 重启服务 → local_rag_node 就能检索到这些文档了

# 下次更新文档后重新入库即可（会自动覆盖同 source 的旧 chunk）
```
